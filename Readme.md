# LLD Design Patterns Practice (Java)

This project contains implementations of essential **Low-Level Design (LLD) patterns** in Java.  
It is organized into clear categories to make learning structured and reusable.

---

## 💡 Why I Started This

While working in my current company, I realized that **understanding LLD deeply is crucial**:

- The Java codebase I work with uses many design principles implicitly
- Knowing patterns makes the architecture easier to understand and extend
- Practicing LLD has improved my overall Java confidence and readability of existing systems
- Helps me think in a cleaner, modular way when writing new features

This repo is my hands-on practice playground for continuous improvement.

---
## 📂 What’s Inside

### 🧱 Creational Patterns
- Factory - Encapsulates object creation
- Builder - Constructs complex objects step-by-step
- Bridge - Separates abstraction from implementation
- Null Object - Eliminates null checks with default behavior
- Object Pool - Reuses expensive objects efficiently

### 🏗 Structural Patterns
- Adapter - Makes incompatible interfaces work together
- Composite - Tree structure of objects with uniform access
- Flyweight - Memory optimization with shared objects
- Proxy - Controlled access to another object

### 🔄 Behavioral Patterns
- Strategy - Swap algorithms at runtime
- Observer - Notify subscribers on state changes
- Decorator - Add features dynamically
- State - Change behavior based on internal state
- Interpreter - Implement language grammar
- Command - Encapsulate actions as objects
- Iterator - Traverse collections uniformly
- Template - Define workflow skeletons
- Visitor - Add operations without modifying objects
- Mediator - Centralized communication between objects
- Chain of Responsibility - Pass request along handlers
- Memento - Save & restore object state
